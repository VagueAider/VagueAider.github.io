VagueAider:

Bridging User-Friendly Vague Intents and Agent-Friendly Executable Instructions for VLM-Based Mobile Agents

Anonymous Authors
Paper Code Dataset Bench

Abstract

A VLM-based mobile agent interprets natural language instructions and directly operates a smartphone. Mobile agents often struggle with vague user instructions that lack explicit application contexts and execution details. Furthermore, we introduce VagueBench, a novel evaluation suite that systematically quantifies the impact of varying instruction ambiguity levels on agent success rates and operational efficiency. Furthermore, we propose VagueAider, a lightweight, training-free, and plug-and-play framework designed to bridge the semantic gap between ambiguous intent and executable GUI actions. Experimental results across VagueBenchmark and external benchmarks demonstrate that VagueAider consistently improves agents’ success rates by a large margin while significantly reducing overhead.

Methodology

Figure 2: Collection of APP Knowledge & VagueAider Framework。

Step 1: Collection of APP Knowledge

Based on an automated pipeline using autonomous agents, we have built a knowledge base covering mainstream applications：

  • App Info Generation: By leveraging the analytical capabilities of LLM combined with the ReAct framework, we can obtain the app's feature descriptions, icons, and core characteristics through websearch tool.
  • Action Path Capture: Employ a Mobile Agent to execute functional instructions, and extract from the execution trajectory a structured path sequence comprising "actions, objects, thoughts, execution, and states."

Step 2: Two-Stage Instruction Rewriting (VagueAider)

1. Intent Expansion

Utilize the common sense reasoning ability of Large Language Models, Expand the user's vague initial intent into a detailed description encompassing specific application contexts and potential operational objectives, complete the missing semantic information.

2. Plan Refinement

Through an Instruction-Aware Refinement Mechanism, Translate the expanded intent into a sequence of atomic, executable GUI instructions, ensure that the Agent can accurately identify target components and execute smoothly.

VagueBench

To explore the sensitivity and robustness of current mobile agents to ambiguous instructions, we propose VagueBench, a set of instructions characterized by three distinct levels of ambiguity

L1

User-Friendly-Ins

Completely vague : it merely expresses user needs and lacks necessary information.

"I want to drink milk tea."

"Fi me a cheaper Switch Toy-Con"

"Check out the latest hit movies and ask ALEX for his opinion."

"Recommend a cost-effective perfume to me and add it to my shopping cart."

L2

Common-Ins

Path Ambiguity: Although a navigation app is specified, no concrete steps are provided; the Agent must explore the path independently.

"Open the Maoyan app to check out the latest movies currently showing, then share them to WeChat to ask friend ALEX for his opinions."

"Use a social media or browser app to find a highly-rated biography, then open an e-commerce app to add it to the cart”

L3

Agent-Friendly-Ins

Clear Expression: Understands user intents, identify the appropriate navigation app for the task and provide detailed step-by-step instructions for both app-inter and app-intra operations.

"First, open a social media app like X or a browser like Google Chrome, tap the search bar, and type 'highly-rated biography' to find recommendations. After identifying a book title, open an e-commerce app like Amazon Shopping, tap its search box, type the book's name, and from the search results, tap the 'Add to Cart' button for the desired item.“
